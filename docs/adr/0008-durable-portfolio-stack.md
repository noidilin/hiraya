# Durable Portfolio Stack for Hiraya Portfolio

- Status: Accepted
- Current architecture: [Portfolio stack](../architecture/portfolio-stack.md)
- Supersedes: none
- Superseded by: none

Hiraya Portfolio is a second public app beside Vintage Storefront and is not deployed to the disposable EKS platform. It lives in a durable Portfolio Stack under `infra/portfolio` so `lazyhiraya.noidilin.dev` and Hiraya Guide remain available when Platform Core and Cluster Bootstrap are destroyed to save cost. This trades away using EKS for the portfolio app in favor of reliable public availability, while Vintage Storefront remains the EKS/GitOps demonstration workload.

The Portfolio Stack owns DNS/ACM, a single CloudFront distribution, private S3 SPA hosting through Origin Access Control, API Gateway HTTP API, Lambda for the Guide API, Bedrock Knowledge Bases backed by Amazon S3 Vectors, a basic Bedrock Guardrail, and related IAM. CloudFront routes `/` to the Vite React SPA and `/api/*` to API Gateway without API response caching; the frontend calls `/api/guide/chat` as a same-origin relative route. The Guide API calls Bedrock Knowledge Bases through `RetrieveAndGenerate`, returns complete answers with normalized citations, refuses when curated evidence is absent, and uses browser-scoped Guide Sessions rather than persistent chat history. Public access is limited through throttling, strict request validation, explicit model token limits, a CloudFront origin-secret header verified by Lambda, and minimal operational logging that excludes prompts, answers, raw retrieved chunks, and request headers.

Curated Project Knowledge lives in `docs/portfolio/` as reviewed Markdown with lightweight frontmatter. Existing report JSON under `docs/reports/` remains upstream evidence only; visitor-facing summaries are ingested instead of raw report data. The knowledge sync workflow stages exactly the six curated Markdown files into a generated directory, mirrors that directory to an S3 knowledge prefix, writes a citation manifest outside the ingested prefix, starts and waits for Bedrock ingestion, and bumps a Lambda knowledge-version environment value so warm Guide API environments refresh cached citation metadata.

Post-acceptance update: the vector store is Amazon S3 Vectors rather than OpenSearch Serverless. Hiraya Guide v1 has tiny, curated Markdown data and does not need hybrid search or advanced filtering, so S3 Vectors reduces expected idle cost and operational surface while preserving Terraform-managed reproducibility. The accepted limitation is semantic-only retrieval unless evaluation data later justifies a richer store.

Project Bootstrap owns the Portfolio Stack backend config and dedicated GitHub OIDC roles. Dedicated Terraform state and roles separate Portfolio Stack plan/apply, portfolio app deploy, and portfolio knowledge sync from existing platform roles so durable public-app permissions do not mix with disposable EKS permissions. Portfolio infrastructure changes use PR plan plus manual approved apply; app and knowledge changes deploy automatically on `main` through a coordinated Portfolio workflow that runs validation, deployment, ingestion, and real Guide smoke tests.
