# Documentation

Start with [`INDEX.md`](INDEX.md) for task-based routing.

## Folder roles

| Folder | Role |
|---|---|
| [`architecture/`](architecture/) | Explains the current system, ownership boundaries, and flows |
| [`runbooks/`](runbooks/) | Gives operational procedures, exact steps, validation, and recovery |
| [`adr/`](adr/) | Records why important architecture decisions were made; start with [`adr/README.md`](adr/README.md) |
| [`references/`](references/) | Holds stable lookup tables such as commands, paths, workflows, and env vars |
| [`portfolio/`](portfolio/) | Curated project knowledge used by Hiraya Guide |
| [`reports/`](reports/) | Canonical report/control data and reporting notes |

## Documentation governance

Use [`references/documentation-guidelines.md`](references/documentation-guidelines.md) before moving or rewriting docs. The short rule is:

> READMEs route, architecture explains, runbooks operate, ADRs justify, references enumerate.
